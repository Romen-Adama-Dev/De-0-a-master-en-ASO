1 Programación de scripts en bash

En esta práctica escribiremos programas, que denominaremos scripts, usando el lenguaje del shell de Linux. Concretamente, vamos a utilizar el bash como hemos venido haciendo hasta ahora, por lo que esta práctica será una continuación de las anteriores.

El principal objetivo de esta práctica es desarrollar la habilidad de escribir scripts a partir de unas especificaciones dadas, usando para ello el lenguaje del bash como un lenguaje de programación completo: variables, condicionales, bucles, funciones, argumentos, etc.

2 Ejercicios sobre sentencias de control

  1. Emplear la orden ls para que, en el caso de que un archivo llamado config exista, muestre el mensaje “config existe”. En caso contrario, debe mostrar el mensaje “config no existe”.

  2. Repetir el ejercicio, pero sin que salgan los mensajes de la orden ls.

  3. Sea la variable var="AAA", escribir el mensaje “itero” mientras la longitud de la cadena sea mayor que 0. En cada iteración debe eliminarse una A de la ristra.

  4. Para una lista de archivos dada (fich1, fich2, fich3,…) mostrar sus permisos.

  5. Dado el valor de una variable “var” que contiene dígitos numéricos, mostrar un mensaje si es de un dígito, otro si es de dos dígitos y otro mensaje si no coincide con ningunos de los patrones anteriores.

  6. Mostrar un menú a un usuario con cuatro posibilidades: (1) listar archivos, (2) mostrar fecha actual, (3) mostrar directorio actual, (4) terminar. Ejecutar la orden correspondiente según la elección del usuario.

3 Ejercicios sobre variables

3.1 Construcción de variables

  1. Definir una variable hello que contenga el literal “hola mundo”.

  2. Definir una variable hithere de forma que contenga el valor de hello.

  3. Definir una variable miruta que contenga la ruta actual.

  4. Crear un archivo de nombre fecha que contenga la fecha actual. Definir la variable ahora y asignarle el contenido del archivo fecha.

  5. Hacer que la variable hithere contenga el literal contenido en hello más la cadena " cruel" (con un espacio a la izquierda).

  6. Mostrar la longitud del texto almacenado en la variable hithere.

  7. Borrar las variables hello y hithere.

3.2 Búsqueda en variables

  1. Inicializar la variable var4 a “AAAABBBabcabcjfkjfkCCC”.

  2. Hacer que la variable var3 contenga la cadena almacenada en var4 salvo la primera “A”.

  3. Hacer que var3 contenga la cadena almacenada en var4 quitando todas las “A” del comienzo.

  4. Hacer que var3 contenga la cadena almacenada en var4 salvo la última “C”.

  5. Hacer que var3 contenga la cadena almacenada en var4 quitando todas las “C” del final.

  6. Siendo la variable var5="/usr/spool/cron", mostrar por pantalla únicamente el nombre del archivo ejecutable.

3.3 Construcción condicional de variables

  1. Mostrar el contenido de la variable DNI; si no existe o está vacía, mostrar el mensaje “no hay un DNI definido”.

  2. Mostrar un mensaje si la variable var4 no está definida. El mensaje sería “var4 no definida”.

  3. Definir la variable var3 y hacer que su valor sea el de la variable var4 si está definida y no es NULL. En caso contrario, que tome como valor el literal “var4 no definida o es null”.

  4. Definir la variable var3 y hacer que su valor sea el de la variable var4 si está definida. En caso contrario, que tome como valor el literal “var4 no definida”.

  5. Definir la variable var3 y hacer que su valor sea el de la variable var4. Si var4 no está definida o es NULL, definirla y hacer que ambas variables tomen el valor “var4 inicializada”.

  6. Definir la variable var3 y hacer que su valor sea el de la variable var4. Si var4 no está definida, definirla y hacer que ambas variables tomen el valor “var4 definida e inicializada”.

  7. Definir la varaible var3 y hacer que su valor sea “var4 definida y distinta de NULL” si la variable var4 está definida y no es NULL.

  8. Definir la variable var3 y hacer que su valor sea “var4 definida” si la variable var4 está definida.

4 Ejercicios sobre desarrollo de scripts

  1. Desarrolle un script que detecte todas las cuentas de usuario cuyo directorio inicial de trabajo no sea descendiente del directorio /home.

  2. Desarrolle un script que detecte aquellos grupos de usuarios que no tienen clave, de forma que se envíe un correo electrónico a los administradores de dicho grupo informándoles de tal situación (use la orden mail para enviarlo). Asuma que la dirección de correo electrónico de los usuarios está almacenada en el campo GECOS del archivo /etc/passwd.

  3. Desarrolle un script al que se le pase una lista de nombres de usuario y nos informe, para cada usuario, si el usuario existe y si su intérprete de órdenes es el bash.

  4. Desarrolle un script al que se le pase una lista de nombres de grupos de usuarios y nos informe, para cada nombre de grupo, si el grupo existe y el número de usuarios que posee.

  5. Desarrolle un script al que se le pase una lista de nombres de usuario y nos proporcione, para cada nombre de usuario, los nombres de los grupos a los que pertenece dicho usuario.

  6. Desarrolle un script al que se le pase una lista de nombres de archivos y nos informe, para cada uno, si existe un archivo con ese nombre en cada uno de los directorios iniciales de los usuarios del sistema.

  7. Desarrolle un script al que se le pase la ruta de un archivo que contiene en cada línea el nombre de un usuario y la ruta completa de un archivo. El script deberá informarnos, para cada línea, si el archivo existe, es propiedad del usuario especificado y tiene permiso de ejecución.

  8. Desarrolle un script al que se le pase una lista de nombres de usuarios. Para cada uno de ellos, debe grabar en el archivo /root/fileinfousuario el usuario, el nombre de su grupo principal y su directorio de trabajo.

  9. Desarrolle un script al que se le pase como un número entero U y encuentre el usuario del sistema que, de todos aquellos que tengan un UID mayor o igual que U, posea el mayor número de archivos bajo su directorio inicial y sus descendientes. El número U debe ser mayor o igual que 500. En caso de encontrarse un usuario, el script imprimirá su nombre, el número total de archivos contabilizados y la ruta inicial desde la que se contabilizó. Ejemplo: “el usuario pepe es propietario de 215 archivo(s) bajo /home/pepe”. En caso de no encontrarse usuarios que cumplan las condiciones del script, se imprimirá un mensaje alusivo a esta incidencia.

5 Ejercicios para practicar

Los ejercicios de esta sección proponen el desarrollo de una serie de scripts con dificultad creciente. Para cada ejercicio se indica el nivel de dificultad que tiene asociado. Se recomienda que primero trabajen los ejercicios con dificultad baja, a continuación aquellos que presentan una dificultad media y por último los ejercicios avanzados con una dificultad alta. Tengan en cuenta que los ejercicios de dificultad alta están por encima de la capacidad media que se exige para aprobar la asignatura. Serían «de notable» o «de sobresaliente».

5.1 Ejercicios con dificultad baja o media

Script 1, dificultad baja. Escribir un script para mostrar solamente las líneas impares de un archivo de texto que se pasa como argumento al script. Posibles ampliaciones: (1) permitir mostrar líneas pares o impares según una opción, (2) permitir un número de archivos variable.

Script 2, dificultad baja. Escribir un script que devuelva todos los usuarios del sistema (campo 1 del fichero /etc/passwd) cuyo shell sea ‘/bin/bash’ (campo 7 del fichero ‘/etc/passwd’) sin utilizar la orden ‘cut’. En su lugar, debes usar las facilidades de expansión de variables del shell.

Script 3, dificultad baja. Escribir un script que permita encontrar aquellos usuarios cuyos UID se encuentran comprendidos en un determinado rango. Los valores de dicho rango se pasan como argumento; el primer argumento debe ser el valor mínimo y el segundo argumento debe ser el valor máximo.

Pista: utiliza la variable IFS para leer el archivo /etc/passwd siguiendo un formato de registro.

Script 4, dificultad media. Escribir un script que permita encontrar aquellos usuarios cuyos UID se encuentran comprendidos en un determinado rango. Los valores de dicho rango se pasan como argumento; el primer argumento debe ser el valor mínimo y el segundo argumento debe ser el valor máximo. Si se omiten los argumentos, el rango se tomará de los valores UID_MIN y UID_MAX que se encuentran definidos en el archivo /etc/login.defs.

Pista: localiza los valores con grep.

Nota: este ejercicio es una versión ampliada del script 3.

Script 5, dificultad media. Escribir un script que, dado un número entero que se pasa como argumento, obtenga la relación de usuarios del sistema que son propietarios de un número de archivos mayor o igual que ese número.

Pista: usa la orden find.

Script 6, dificultad media. Escribir un script que permita buscar todos los archivos con extensión .cpp y renombrarlos como .cc. La búsqueda se hará recursivamente a partir de una ruta inicial que vendrá dada como argumento al script. Si se omite el argumento, la ruta inicial será el directorio actual.

Pista: usa basename o emplea los operadores de cadenas del bash.

Script 7, dificultad media. Escribir un script que permita buscar todos los archivos con extensión .doc o .docx y normalizar sus nombres, de manera que todos sus caracteres sean letras minúsculas (incluyendo la extensión). Por ejemplo, el archivo MiFichero.DOCX debe pasar a ser miarchivo.docx. La búsqueda se hará recursivamente desde una ruta inicial que vendrá dada como argumento al script. Si se omite el argumento, la ruta inicial será el directorio actual.

Por cada normalización ejecutada, se añadirá una línea al final del archivo de registro /var/log/cambiosDoc que tendrá:

  • La fecha y hora del cambio en formato RFC 3339 con precisión de segundos. Consulta la página de manual de la orden date para ver cómo obtener la fecha en ese formato.
  • Un tabulador.
  • La ruta original del archivo.
  • Un tabulador.
  • El nuevo nombre del archivo (sólo el nombre, sin el directorio).

Ejemplo de línea: 2013-10-16 10:29:43+01:00 /home/pepe/MiFichero.Doc miarchivo.doc

Pista: aprovecha la orden tr.

Script 8, dificultad media. Escribir un script que reciba dos argumentos: la ruta de un directorio de origen ORG y la ruta de un directorio de destino DST. El script deberá enlazar en DST todos los ficheros que haya en ORG, formando una estructura de carpetas organizadas por años. En la carpeta del año A estarán los enlaces simbólicos hacia todos los archivos de ORG cuya fecha de última modificación sea del año A. Solo se crearán carpetas para los años que tengan ficheros asociados. Se puede dar una especificación con recursividad o sin ella. Posibles ampliaciones: (1) crear un siguiente nivel de carpetas con los meses del año, por ejemplo DST/2020/11/fichero, (2) añadir un siguiente nivel de carpetas con los días.

Script 9, dificultad media. Escribir un script que tome como argumentos dos archivos F1 y F2 e imprima su contenido en líneas entrelazadas alternativamente: primero escribirá la primera línea de F1, luego la primera línea de F2, luego la segunda línea de F1, la segunda línea de F2 y así sucesivamente. Si uno de los dos archivos tiene menos líneas que el otro, cuando se acabe de mostrar el más corto el script escribirá el resto del contenido del más largo.

5.2 Ejercicios avanzados con dificultad alta

Script 10, dificultad alta. Escribir un script que tome como argumentos dos o más archivos e imprima su contenido en líneas entrelazadas alternativamente: primero escribirá la primera línea de cada archivo, siguiendo el orden en el que han sido indicados como parámetros, luego la segunda línea de cada archivo y así sucesivamente. Si se acaban las líneas de uno de los archivos, el script seguirá escribiendo líneas del resto de archivos de forma entrelazada y continuará de esta manera hasta que termine con todos los archivos.

Nota: este ejercicio es una versión generalizada del script 9.

Script 11, dificultad alta. Escribir un script que dadas dos carpetas A y B, que se pasarán como argumentos al script, muestre qué archivos están presentes solamente en una de las dos carpetas. Para cada archivo que cumpla esa condición, se mostrará su ruta completa (A/archivo o B/archivo, según el caso). El rastreo de archivos se hará sólo en el primer nivel de profundidad de las carpetas, no se hará recursivamente. Sólo se comparará el nombre de los archivos, no su tipo ni su contenido.

Script 12, dificultad alta. Escribir un script que compruebe que los shells asignados a los usuarios del sistema (campo 7 del archivo /etc/passwd) aparecen en el archivo /etc/shells. Si el shell de algún usuario no aparece en ese archivo, el script debe escribir el nombre del shell en cuestión y debe continuar con la comprobación. El resultado final del script debe ser una lista (probablemente con elementos repetidos) de todos los shells asignados a los usuarios que no aparezcan en el archivo /etc/shells.

Script 13, dificultad alta. Escribir un script que reciba como parámetros tres valores numéricos X, G y P. El script detectará, cada X minutos, aquellos usuarios del sistema cuyo grupo principal tenga el GID G y sean propietarios de más de P procesos en el sistema.

Pista: utiliza la orden ps para seleccionar aquellos procesos cuyo propietario sea un usuario dado.
